# Structure des Dossiers

Découvrez la structure des dossiers de ce modèle et son contenu.

```
|
├─ app                     // Contient les routes et la logique de l'application
│  ├─ (dashboard)          // Groupe les pages liées au tableau de bord
│  │  ├─ (apps)            // Applications spécifiques
│  │  ├─ (chart)           // Pages et composants de graphiques
│  │  ├─ (components)      // Composants réutilisables spécifiques au tableau de bord
│  │  ├─ (diagrams)         // Pages et composants de diagrammes
│  │  ├─ (forms)           // Pages et composants de formulaires
|  |  ├─ (home)            // Page d'accueil du tableau de bord
│  │  ├─ (icons)           // Icônes spécifiques
│  │  ├─ (invoice)         // Pages et composants de facturation
|  |  ├─ (map)             // Pages et composants de cartes
│  │  ├─ (tables)          // Pages et composants de tableaux
│  │  ├─ blank             // Page vierge ou modèle
|  |  ├─ diagram           // Page de diagramme (peut-être une page spécifique au-delà des (diagrams) génériques)
│  │  |-layout.jsx         // Layout spécifique au groupe de routes (dashboard)
│  │  |-main-layout.jsx    // Layout principal du tableau de bord
│  │  
│  ├─ api                   // Points d'API (API Routes)
│  ├─ assets                // Fichiers statiques comme des images, des vidéos, etc.
│  ├─ auth                  // Pages et logique d'authentification
│  ├─ error-page            // Pages d'erreur personnalisées
│  ├─ utility               // Fonctions utilitaires diverses
│  |- favicon.ico          // Icône de favori pour le site
│  |- layout.jsx           // Layout principal de l'application
│  |- not-found.js         // Page 404 personnalisée
│  |- page.jsx             // Page d'accueil racine de l'application
│  
├─ components              // Composants globaux réutilisables à travers l'application
│  ├─ auth                  // Composants liés à l'authentification
│  ├─ landing-page          // Composants pour la page d'accueil/d'atterrissage
│  ├─ partials              // Petits morceaux de UI ou sections
│  ├─ svg                   // Composants SVG ou icônes
│  ├─ task-board            // Composants pour un tableau de tâches
│  ├─ ui                    // Composants d'interface utilisateur génériques (boutons, inputs, etc.)
|  |-blank.jsx             // Composant vierge ou modèle
│  |-dashboard-select.jsx  // Composant de sélection pour le tableau de bord
│  |-dashboard-dropdown.jsx// Composant de menu déroulant pour le tableau de bord
│  |-date-picker-with-range.jsx // Composant de sélecteur de date avec plage
│  |-delete-confirmation-dialog.jsx // Composant de boîte de dialogue de confirmation de suppression
│  |-error-block.jsx       // Composant pour afficher des messages d'erreur
│  |-header-search.jsx     // Composant de barre de recherche d'en-tête
│  |-layout-order.jsx      // Composant lié à l'ordre de la mise en page
│  |-ripple.jsx            // Composant d'effet "ripple"
|
├─ config                  // Fichiers de configuration de l'application
├─ hooks                   // Hooks React personnalisés
├─ lib                     // Bibliothèques utilitaires et fonctions
│  ├─ docs                 // Utilitaires spécifiques à la documentation
│  |-appex-chat-options.jsx// Options pour les graphiques ApexCharts
│  |-auth.js               // Fonctions utilitaires d'authentification
│  |-utils.js              // Fonctions utilitaires générales
│  
├─ pages                   // Dossier pour les pages traditionnelles de Next.js (si Next.js Pages Router est utilisé en parallèle de App Router)
│  ├─ docs                 // Pages spécifiques à la documentation
│  |-_app.jsx              // Fichier Next.js _app.js pour l'initialisation de l'application
│  |-_meta.json            // Métadonnées pour les pages (souvent utilisé par Nextra)
│  
├─ provider                // Fournisseurs de contexte React (ex: thème, état global)
├─ public                  // Fichiers statiques accessibles publiquement (images, police, etc.)
├─ store                   // Gestion de l'état global (ex: Redux, Zustand)
|- .env.local              // Variables d'environnement locales
|- .gitignore              // Fichiers et dossiers à ignorer par Git
|- jsconfig.json           // Configuration JavaScript (pour VS Code notamment)
|- middleware.js           // Middleware Next.js
|- next.env.d.ts           // Définitions de types pour les variables d'environnement Next.js
|- next.config.js          // Configuration de Next.js
|- package-lock.json       // Verrouillage des dépendances npm
|- package.json            // Informations sur le projet et dépendances
|- README.md               // Fichier README du projet
|- postcss.config.js       // Configuration de PostCSS
|- tailwind.config.js      // Configuration de Tailwind CSS
|- theme.config.js         // Configuration du thème (souvent lié à Nextra)
|- yarn.lock               // Verrouillage des dépendances Yarn
```
