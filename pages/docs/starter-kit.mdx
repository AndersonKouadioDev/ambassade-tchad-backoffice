# Kit de Démarrage

Le Kit de Démarrage est un modèle minimal à partir duquel vous pouvez démarrer votre projet rapidement, au lieu de devoir supprimer des éléments inutiles. Voici quelques points rapides sur le Kit de Démarrage :

  * Pas d'authentification
  * Pas de tableau de bord
  * Pas d'applications
  * Pas de pages (spécifiques à la démo)
  * Pas d'éléments (spécifiques à la démo)

-----

### Structure des Dossiers

Découvrez la structure des dossiers du Kit de Démarrage et son contenu.

```
|
├─ app                     // Contient les routes et la logique de l'application
│  ├─ (dashboard)          // Groupe de routes pour le tableau de bord (même si vide dans le starter)
│  │  |-layout.jsx         // Layout spécifique au groupe (dashboard)
│  │  |-main-layout.jsx    // Layout principal du tableau de bord
│  │  |-page.jsx           // Page d'accueil du tableau de bord
│  │
│  ├─ api                   // Points d'API (API Routes)
│  ├─ assets                // Fichiers statiques comme des images
│  |- favicon.ico          // Icône de favori pour le site
│  |- layout.jsx           // Layout principal de l'application
│  |- not-found.js         // Page 404 personnalisée
│
├─ components              // Composants globaux réutilisables à travers l'application
│  ├─ landing-page          // Composants pour la page d'accueil/d'atterrissage
│  ├─ partials              // Petits morceaux de UI ou sections
│  ├─ svg                   // Composants SVG ou icônes
│  ├─ ui                    // Composants d'interface utilisateur génériques
|  |-blank.jsx             // Composant vierge ou modèle
│  |-dashboard-select.jsx  // Composant de sélection pour le tableau de bord
│  |-dashboard-dropdown.jsx// Composant de menu déroulant pour le tableau de bord
│  |-date-picker-with-range.jsx // Composant de sélecteur de date avec plage

├─ config                  // Fichiers de configuration de l'application
├─ hooks                   // Hooks React personnalisés
├─ lib                     // Bibliothèques utilitaires et fonctions
│  ├─ docs                 // Utilitaires spécifiques à la documentation
```

### Configuration du Thème

La configuration des thèmes dans Dashcode est incroyablement simple. Le fichier de configuration principal est `use-config.ts`. Dans le fichier `use-config.ts`, vous pouvez configurer votre thème, votre mise en page, le type de barre latérale, la couleur de la barre latérale, et bien plus encore. Référez-vous à l'extrait de code ci-dessous pour toutes les options disponibles.

```ts filename="hooks/use-config.ts"
export const Config = {
  "collapsed": false,       // Indique si la barre latérale est repliée par défaut
  "theme": "violet",        // Thème de couleur principal de l'application (ex: "violet", "rose", "gray")
  "skin": "default",        // Type de "skin" ou style visuel ("default" ou "bordered")
  "layout": "vertical",     // Type de mise en page ("vertical", "horizontal", etc.)
  "sidebar": "draggable",   // Comportement de la barre latérale ("draggable" ou autre)
  "menuHidden": false,      // Indique si le menu est caché
  "showSearchBar": true,    // Affiche la barre de recherche
  "topHeader": "default",   // Style de l'en-tête supérieur ("default" ou "links")
  "contentWidth": "wide",   // Largeur du contenu principal ("wide" ou "boxed")
  "navbar": "sticky",       // Position de la barre de navigation ("sticky", "fixed", "default")
  "footer": "sticky",       // Position du pied de page ("sticky", "default", "hidden")
  "isRtl": false,           // Active ou désactive le mode de lecture de droite à gauche (Right-to-Left)
  "showSwitcher": true,     // Affiche le commutateur de thème/mode
  "subMenu": false,         // Indique si un sous-menu est actif
  "hasSubMenu": false,      // Indique si la mise en page a un sous-menu
  "sidebarTheme": "redwood", // Thème de couleur spécifique de la barre latérale
  "headerTheme": "light",   // Thème de couleur spécifique de l'en-tête
  "radius": 0.5             // Rayon des bords des éléments (en rem)
};
```
